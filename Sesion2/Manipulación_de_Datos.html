Manipulación de datos con dplyr y tidyr

Por qué usar dplyr y tidyr?

  • Rapidez - dplyr y tidyr transforman la base de datos de manera rápida y eficiente computacionalmente.
  • Legibilidad - La sintaxis es directa y de fácil lectura.
  • Encadenamiento - Múltiples instrucciones dentro de una misma estructura
  • Pueden ser usados para manipular fuentes externas sin necesidad de cambiar de lenguaje.

Tidy Data

En general, es deseable tener las bases de datos en formato "tidy".

En conjuntos "tidy:

  • Cada variable forma una columna
  • Cada observación es una fila
  • Cada tipo de unidad observacional forma una tabla

Dplyr

dplyr tiene el objetivo de ser una gramática para la manipulación de datos. Se definen los siguientes "verbos":

  • filter() filtros basados en criterios lógicos
  • select() Selección de columnas
  • arrange() Ordenar las filas según alguna variable
  • rename() renombrar columnas
  • group_by() Agrupa los datos según variables en común para realizar cálculos
  • mutate() Crear una nueva variable
  • summarize() sumarizar valores a una sola fila por grupo

Donde los nombres de las variables se usan sin comillas.

Encadenamiento de Funciones

El paquete dplyr hereda el operador %>% llamado "pipe", permie encadenar funciones pasando como primer parámetro a la funcion escrit a su derecha el resultado de la funcion al lado izquierdo.

# summarize(data, new_var1=[expression], new_var2=[expression])
new_data <- iris %>% 
  filter() %>% 
  select() %>%
  mutate() %>% 
  arrange() %>% 
  rename() %>% 
  group_by() %>% 
  summarize() %>%

Encadenamiento de Funciones

Con el encadenamiento de variables es posible reducir la cantidad de codigo escrita para una maniuplacion de datos. Por ejemplo se podria pasar de

res1 <- filter(flights, origin=="JFK")
res2 <- mutate(res1, speed=distance / (air_time), Distancia=ifelse(distance < 700, "Short", "Long"))
res3 <- select(res2, origin, dest, distance, speed, Distancia)
res4 <- arrange(res3, desc(speed))
res5 <- group_by(res4, origin, dest,distance, Distancia)
res6 <- summarise(res5, MeanSpeed=mean(speed, na.rm = T))

Encadenamiento de Funciones

o esto

Forma2 <- summarise(
  group_by(
    arrange(
      select(
        mutate(
          filter(
            flights, origin=="JFK"),
          speed=distance / (air_time),
          Distancia=ifelse(distance < 700, "Short", "Long")), 
        origin, dest, distance, speed, Distancia), desc(speed)), 
    origin, dest,distance, Distancia), MeanSpeed=mean(speed, na.rm = T))

Encadenamiento de Funciones

A una escritura muchos legible y limpia:

Forma3<- flights %>% 
  filter(origin=="JFK") %>% 
  mutate(speed=distance / (air_time), Distancia=ifelse(distance < 700, "Short", "Long")) %>% 
  select(origin, dest, distance, speed, Distancia) %>%  
  arrange(desc(speed)) %>% 
  group_by(origin, dest,distance, Distancia) %>%
  summarise(MeanSpeed=mean(speed, na.rm = T))

Tidyr

Los verbos de tidyr son:

  • gather() y spread() convierten la base de datos en formato ancho o largo
  • separate() y unite() separa una única columna en multiples columnas o visceversa
  • complete() convierte los valores ausentes implicitos en valores explicitos completando las combinaciones ausentes.

las funciones de tidyr pueden ser encadenadas junto con las funciones de dplyr

Gather y spread

Permite transponer filas en columnas o visceversa:

library(tidyr)
messy <- data.frame(
  name = c("Wilbur", "Petunia", "Gregory"),
  a = c(67, 80, 64),
  b = c(56, 90, 50)
)
messy
##      name  a  b
## 1  Wilbur 67 56
## 2 Petunia 80 90
## 3 Gregory 64 50

Gather y spread

messy %>%
  gather(drug, heartrate, a:b)
##      name drug heartrate
## 1  Wilbur    a        67
## 2 Petunia    a        80
## 3 Gregory    a        64
## 4  Wilbur    b        56
## 5 Petunia    b        90
## 6 Gregory    b        50

En la documentación de tidyr se encuentran ejemplos y aplicaciones de las funciones separate(), unite() y complete()
